# 장기 시뮬레이션 — 100명 × 50R 현상 전개 (2026-05-29)

5R 단발 시뮬은 "현상 형성 전" 이라(보고서·OASIS 자인) 정보확산·양극화·herd 의 **시간
전개**를 볼 수 없었다. 같은 ontology(e2e2, seed 7)로 라운드만 5→50 으로 늘려 현상이
어디로 수렴하는지 측정한다 — 인원·페르소나를 고정하고 라운드만 바꾼 변수 통제 비교.

## 셋업

- ontology: `runs/e2e2-2026-05-29/ontology_a_persona.json` (100명, seed 7 — 5R baseline 과 동일 출발점)
- rounds 50, model `openrouter/qwen/qwen-plus`, batch 20, semaphore 10, `--token-budget 10_000_000`
- 완주: `early_exit=False`, `rounds_run=50`, tokens 9.10M, **100명 전원 활동**, 1891 events
- 후반 R40-49 `DO_NOTHING` 평균 2.6% — budget 압박/행동 위축 없음 (깨끗한 데이터)

> 첫 시도(`runs/longrun-100x50R-2026-05-29/`)는 기본 token-budget 3M 에 R18 에서 걸려
> early_exit 했다. 행동 위축은 없었으나(DO_NOTHING 0~7%) 36% 만 봐서 budget 10M 으로 재실행.

## 누적 현상 추세

R0 부터 누적 라운드를 늘려가며 측정 (`DataAggregator.aggregate_events`, LLM 무관·결정적):

(`cascade_max_depth` 는 전 구간 2 로 고정이라 표에서 생략.)

## 발견

### 1. 양극화는 echo chamber 로 수렴하지 않는다

`ideology_assortativity` 가 +0.052(5R) → -0.018(R0-9) → -0.135(R0-19) 로 **음수 전환 후
-0.11 ~ -0.13 에서 안정**한다. 양수(동질 선호 = echo chamber)로 가지 않고 약한
disassortative(반대 성향과도 연결)로 수렴. `follow_ideology_gap` 도 0.18→0.23 으로 follow
상대의 이념 거리가 오히려 벌어진다. OASIS 가 보고하는 "규모↑ → 양극화 심화" 와 대비되는
동학이다.

### 2. herd 집중은 초기 피크 후 완화된다

`popularity_gini` 가 R0-9 에서 0.606 으로 피크 친 뒤 **단조 감소해 0.462(50R)**. 
`early_mover_share` 는 0.804 → **0.306 으로 급감** — 초기 인기 노드가 후반 FOLLOW 를
흡수하는 비율이 크게 떨어진다. 즉 "부익부(rich-get-richer)" 가 지속되지 않고 **인기가
재편**된다.

### 3. 확산은 넓어지되 깊어지지 않는다

`cascade_max_scale` 12→34, `n_cascades` 5→14 로 캐스케이드 규모·개수는 늘지만
`cascade_max_depth` 는 2 에 고정. 재게시의 재게시(깊은 바이럴)는 안 생기고 넓고 얕은
확산에 머문다 (REPOST 5~6% 수용 한계, `docs/qa/metrics.md`).

## 해석과 한계

- **게이트 설계 영향 가능성**: ActionSelector 의 behavior_tendency 게이트는 "아직 follow
  안 한 author" 만 FOLLOW 후보로 두고 이미 follow 한 author 재-FOLLOW 를 거부한다(#140).
  이 정책이 인기 노드 독점(early_mover↑)을 구조적으로 억제해 herd 완화·disassortative
  수렴에 기여했을 수 있다 — 위 동학은 순수 창발이 아니라 게이트 정책과 얽혀 있다.
- **단일 ontology**: seed 7 한 모집단의 50R. 다른 ontology·seed 에서 같은 추세인지는
  미검증. self-baseline(`docs/qa/baseline.md`)은 5R 분포라 50R 누적과 직접 비교 불가.
- **OASIS 직접 비교 아님**: OASIS 가 단일 베이스라인 수치를 공개하지 않아(`metrics.md`)
  정성 대비만 가능하다.

## 재현

```bash
litemiro-run \
  --ontology-a runs/e2e2-2026-05-29/ontology_a_persona.json \
  --ontology-b runs/e2e2-2026-05-29/ontology_b_memory.json \
  --rounds 50 --output-dir runs/longrun-50R-2026-05-29 \
  --llm-model openrouter/qwen/qwen-plus \
  --batch-size 20 --semaphore-limit 10 --token-budget 10_000_000
```

events 는 `runs/` (gitignore) 라 위 셋업으로 재현한다 — seed 7 고정이라 같은 궤적.
누적 추세 표는 `DataAggregator.aggregate_events(events[:round], ideology)` 를 R0-9/19/29/39/49
로 잘라 산출했다.
